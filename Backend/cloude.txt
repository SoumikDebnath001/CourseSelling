The core system will comprise three distinct layers: a foundational layer, designated as "Foundation Level"; an intermediate layer, termed "Tier One" or "Level One"; and an advanced layer, referred to as "Tier Two" or "Level Two." All members admitted or logged into the system will automatically qualify for Foundation Level courses.
Eligibility for Tier One courses requires meeting two specific criteria. For instance, in a course structured with Foundation, Tier One, and Tier Two levels, such as a fast bowling course, a participant must successfully complete the Foundation Level before progressing to Tier One (or Level One, as it will be referred to hereafter).
Additionally, the system will offer miscellaneous courses, the categorization of which will be determined by the administrator. The admin dashboard will feature a dedicated field for assigning a level (Foundation, Level One, or Level Two) to each course.
Within a progressive course, participants advance sequentially from Foundation to Level One, and then to Level Two. Miscellaneous courses, if they align with the participant's current highest level and category, will also be accessible. For example, a participant at Level Two in a batting course would be eligible for a Level Two batting-related miscellaneous course.
New functionalities will be introduced in the admin panel, allowing administrators to designate a course as either "Progressive" or "Miscellaneous" via a toggle switch. If a course is marked as "Progressive," an additional option, such as a slider, will enable the administrator to specify the highest attainable level within that course (e.g., Level One or Level Two).
A point system will be implemented, visible exclusively to users, enabling them to track their progress through analytics, including accumulated points and completed courses. When creating a course, administrators will define the point value for each module and, further, for each topic within a module. To streamline this process, an "Apply for All" option will be available. After an administrator inputs points for a single module, this option can be selected to apply the same value proportionally to all other modules. A similar "Apply for All" functionality will exist for topics. Administrators will retain the ability to manually adjust these point values after initial application.
A progress bar on the user's home screen will consistently display the points remaining to reach Tier Two within a specific course. If a user is enrolled in multiple courses, a card-changing system will be implemented, automatically cycling through progress updates for each course. Furthermore, users will have a dedicated button to manually switch between course progress displays.
